Introduction to Web APIs

What is a Web API?

Web APIs, or web application programming interfaces, are incredibly important in the world of software development. They are what allows a website:

  • to find the closest Tim Hortons to you,
  • to remember what you added to your shopping cart, and
  • to save your login information.

Web APIs essentially act as bridges that allow different software applications or websites to communicate and work together seamlessly by sharing information over the internet.

How does it work?

Restaurant Analogy

To help make APIs and the information sharing process easier to understand, imagine that you're a customer at a restaurant.

  • When ordering, you need to place an order by communicating with the waiter/waitress.
  • When the waiter/waitress takes the order, they need to write it down on their notepad, often using a shorthand form of the menu items, which the chef will still be able to understand.
  • They will then give that order to the chef to tell them what you want to eat.
  • Then the chef will create your order, and the waiter/waitress will take your food to you.

In this scenario, you — the customer — are the client.

  • The chef would be the server,
  • the waiter/waitress would be the API,
  • the order would be the request, and
  • the food would be the response.

Notice how the API — or the waiter/waitress in this analogy — is the intermediary (also known as the middleman), between the user and the server.

The API is responsible for delivering the request — or the customer's order — from the user to the server, and for delivering the response — or the customer's food — from the server to the user.

Actual API Process

APIs work by following a set of rules — or protocols — that define how the communication between applications should happen. One of the most common protocols for web APIs is HTTP, which stands for HyperText Transfer Protocol.

HTTP defines how a client (the application that makes a request) and a server (the application that responds to the request) should exchange messages. The messages include information such as the URL (the web address where the request is sent), the method (the type of action the client wants to perform), the headers (additional information about the request or the response), and the body (the data that is sent or received).

When a user wants to access some information or service from a website, they send a request to the website's server. The request contains some details about what the user wants, such as the URL (Uniform Resource Locator) of the website, as well as the method (such as GET, POST, PUT, or DELETE) and the parameters (such as keywords, filters, or options) of the request.

The website's server receives the request and processes it. The server may need to access some data from a database or perform some calculations or operations to fulfill the request. The server then creates a response that contains the information or service that the user requested.

So how do APIs fit into all of this? When using a web API, it acts as the intermediary, or the middleman, between the user and the server. The web API follows some rules or standards to ensure that the communication is clear and consistent. These rules or standards are called the protocol of the web API. The protocol defines how the request and response should be formatted, what kind of data should be included, and how to handle errors or exceptions.

The user receives the response from the web API and displays it on their browser or application. The response may contain some data in a JSON (JavaScript Object Notation) format, which is a common way to represent structured data in a human-readable and machine-readable way. The response may also contain some status codes that indicate the success or failure of the request, such as 200 (OK), 404 (Not Found), or 500 (Internal Server Error).

Web API examples

There is a wide variety of web APIs, including Fetch, Web Storage, and — the one we're covering today — Geolocation. They all have different purposes, where:

  • Fetch is used for fetching data from the internet in web applications, and
  • Web Storage is a simple way to store and remember small pieces of information on your device.

Geolocation

What is Geolocation API?

Geolocation APIs enable developers to access a device's geolocation, it utilizes various methods like WIFI, GPS, and IP to determine a user's location, and returns latitude/longitude coordinates and a radius indicating result accuracy. Geolocation APIs are pivotal elements for creating features such as location-based services, maps, navigation, and local search, which leads to personalized content and directions. This functionality is also essential in areas like emergency services, asset tracking, and IoT devices.

Pros & cons

Pros

  • Weather Reports: With advanced technology like satellites, weather forecasts have become more accurate, aiding in better preparation for changing weather conditions. Geolocation APIs allows users to access live local weather information just a click away.

  • Content Personalization for Social Media: Geolocation APIs empower businesses to deliver tailor-made marketing offers. Based on users' different background, interests, motivations, and goals, it allows businesses to deliver the right message to the right person at the right time in the right form, which enhances the user experience through personalized content on social media platforms.

  • Marketing and Customer Engagement: Geolocation provides businesses with valuable data to help them better analyze customer behaviours, solve problems, discover new opportunities, and optimize their business processes.

  • Employment and Business Networking: Job listing websites utilize geolocation to connect job seekers with opportunities in their vicinity, facilitating localized job searches.

Cons

  • Stalking: A significant drawback of geolocation APIs involves the potential for inadvertent disclosure of one's location on social media, making individuals vulnerable to stalking and real-life attacks by those with malicious intent.

  • Concerns and Data Rights: With the growth of privacy awareness, according to Matthew Crozier (n.d.)
    “Jurisdictions leading the way on data privacy like Europe and Canada, along with some US Federal Statutes such as COPPA (Children's Online Privacy Protection Act) are starting to form an opinion that IP addresses are PII, or "personal data" (Matthew Crozier, n.d.).
    Gathering such information without explicit consent may pose legal and ethical challenges,emphasizing the need for caution in data collection.
